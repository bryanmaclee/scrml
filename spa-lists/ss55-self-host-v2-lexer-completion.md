**THE WORKAROUND CATALOG (MANDATORY — these are LIVE compiler bugs the slices hit; using the workaround is required or you re-hit them; committed in `progress.md` F1-F9):**
- **F1** — library mode (`emit-library.ts`) can't lower typed-payloads/`match`/payload-variant → WRAP the lexer in `<program>` + consume the emitted client.js (the browser path routes through the real emitter). *(This is the Road-B strategic blocker for the PARSER wave's cross-module imports — but the lexer stays single-file under `<program>`, so it does NOT bite here.)*
- **F2** — int-literal `match` arms drop silently → dispatch operators on the 1-CHAR STRING (`match ch { "=" :> … }`), never on `peekCode` ints.
- **F4** — a bare variant resolves ONLY in fn-return position or qualified → use qualified `TokenKind.X` + typed-return helper fns exclusively (never bare `.Variant` as a fn-arg / struct-field / array-element).
- **F5** — a `match` subject must be typed (host-method returns are `asIs`) → annotate (`const ch: string = …`).
- **F6** — `match`-arm `|` ALTERNATION drops silently for string/payload alternatives → single-literal arms (lookup table) or an `is .Variant` chain; nullary-enum alternation (`.A | .B :> x`) is the ONLY safe alternation.
- **F8** — reassign-to-anonymous-struct poisons a string field's host-method return type across fns → factor per-char typed-param helpers with return annotations (never reassign a `let` to an anon struct literal).
