## Parked / escalated to PA
1. **`api-fetch-decode-roundtrip` (runtime) — BLOCKED, parked.** §60's runtime half
   (emitted `fetch(base+path)` → `parseVariant` decode → `<request>.data`/`.error`) is
   untestable through the current adapter: `conformance/adapters/impl1-ts.ts` only installs
   a server-dispatch fetch mock that routes `/_scrml/*` (the §52 server-fn path). An `<api>`
   endpoint fetches a foreign external URL (`base + path`, e.g. `https://api.example.com/users/1`),
   which no mock intercepts → a real network call / hang under happy-dom. The residual
   **adapter gap**: add a base+path-keyed external-URL fetch mock (declared per-case, e.g. a
   new `apiStub` verb keyed by endpoint name → response body) so the emitted fetch resolves
   deterministically and the parseVariant decode + `.data`/`::ParseError` routing become
   observable. This is an adapter/contract extension (PA/design call — parallels the
   `serverStub`/`serverDb` verbs), NOT buildable in this list. Filed here + in the ping.
