# domain.map.md
# project: scrml
# updated: 2026-07-19T21:52:34-06:00  commit: df2ac831

scrml is a single-file full-stack language + compiler (not a web app with a runtime business domain). "Domain concepts" here are the language's own primitives, normatively defined in `compiler/SPEC.md` (¬ß1-¬ß65+). This map is a navigation index into that spec, grouped by concern βÄî not a restatement of the normative text.

## Core Concepts (by SPEC section)

**Reactivity** βÄî `@cell` reactive declarations (¬ß6, V5-strict access model); a cell auto-subscribes every read site. Value-native maps/sets (¬ß59) give `@cell:[K]V` / `set[K]` first-class reactive collection types. ¬ß6.6.9: server-fn / client-cell read βÄî "THE SPLIT" βÄî a server function reading a client cell gets an explicit CPS-marshal boundary (E-REACTIVE-003 + W-SERVER-DERIVED-MARSHAL) instead of a silent value smuggle.
**State machines** βÄî `<engine for=Type>` (¬ß51) governs variant-graph progression via `rule=`/`initial=`/`<onTransition>`/`<onTimeout>`/`<onIdle>`; `<engine server=@source>` gives server-authoritative hydration (¬ß52.4.4). Sibling: ¬ß54 nested substates. 44 E-ENGINE-* diagnostic codes (largest single family βÄî see error.map.md).
**Client Router / soft navigation βÄî ¬ß20.8, LANDED.** `<program>` is the persistent application shell (single-file `<page>` children or multi-file `pages/*.scrml`); it MAY contain exactly one `<outlet>` βÄî the region into which the current route's SSR-fetched content swaps on navigation. ¬ß20.8.3 link-boost (a delegated document-level `click` listener in `compiler/src/runtime-template.js`, boot-wired only on an app with an `<outlet>`) intercepts internal same-origin cross-page `<a href>` clicks and routes them through the soft-nav engine instead of a full reload; `hard` opts a specific link out.
**Standalone tools βÄî ¬ß64** βÄî `<program kind="tool">` compiles to a CLI-style module with one `function main(args)` entry (E-TOOL-001..006). `<program kind="tool" serve=PORT>` emits a compiler-owned `Bun.serve` harness hosting the tool's `<endpoint>`/SSE routes headlessly (no CSRF, no cookie-session, no SSR); cookie-session `auth=` on a `serve=` tool is fail-closed rejected (E-TOOL-SERVE-AUTH-UNSUPPORTED). `<foreign lang="ts">` (¬ß23.6) gives a library file its own foreign-language declaration.
**Pattern matching / enums** βÄî `match`/`is` over closed enum unions (¬ß18); shorthand `.Variant` forms (¬ß14.5); exhaustiveness is a compile error, not a runtime default. E-TYPE-082 fail-arity ruling for enum-variant construction payload arity.
**Absence** βÄî `not` is the ONE canonical absence value (¬ß42). `null` and `undefined` are NOT valid scrml tokens in ANY position (expression/attribute/type/identifier) βÄî hard rule, W-ABSENCE-IN-SCRML-SOURCE lint + E-SYNTAX-042 hard error. Defined-but-empty values (`""`, `0`, `false`, `[]`, `{}`) are NOT absence.
**Logic contexts** βÄî `${}` (logic), `?{}` (SQL), `#{}` (CSS), `_{}` (foreign/escape-hatch, ¬ß23), `^{}` (meta/compile-time eval, ¬ß22), `~{}` (test, ¬ß19.13), `!{}` (error-arm, ¬ß19). Each is a distinct parse context (¬ß3-¬ß4, ¬ß7-¬ß9).
**SQL** βÄî `?{}` inline SQL blocks (¬ß8) resolve against `<db>`/`<schema>` (¬ß39); `?{}` in a library context emits reactive-deps-aware client SQL (W5b). E-SCHEMA-001/002/004 + W-SCHEMA-001 strict ¬ß39.4 `<schema>` column-type checks wired; a real-DB conformance adapter (Bun.SQL in-memory seam, `sqlEngine` opt-in) exercises live-DB behavior in the D3 corpus.
**CSS βÄî the ¬ß65 scrml-native model βÄî Wave-1 EMISSION LANDED.** Deletes cascade specificity. `<theme>` (program-scope named-token block, lowers to ¬ß25 CSS custom properties) + `<defaults>` (app-wide bare-element defaults, the "base" @layer) are structural elements (NOT HTML; attribute-registry.js:503/516). `:where()`-flat emission on scoped `#{}` (unconditional arms), the `@layer reset, global;` order + a built-in default-ON `reset` layer (opt-out `<program reset="none">`), `<theme>` token βÜ£ `:root` custom-property lowering via a `@`-sigil use-site syntax (a token DECLARATION stays bare; a use-site REFERENCE requires `@`, disambiguating from the ¬ß25 reactive-CSS-var cell bridge and from a literal CSS keyword), and the ¬ß65.6 runtime theme-switch reflection are all implemented in `emit-css.ts` + `emit-theme-reset.ts` + `emit-client.ts`. Diagnostic: `E-THEME-TOKEN-UNKNOWN`. Remaining Wave 2-3 work: the `style`-value type + dispatch, `<defaults>` as style-as-value, the full `@layer` chain + Tailwind-in-`utilities` integration, `--explain-style`.
**Realtime βÄî ¬ß38.13 `<channel watches=table>`** βÄî a change-feed-over-external-DB-writes primitive, distinct from the general ¬ß38 WebSocket `<channel>`. Front-end recognition + `RowChange` synthesis (`channel-watches.ts`) + Postgres trigger DDL install + the bundled-`pg` LISTEN bridge + client `__change` frame dispatch are all landed. `<onchange>` is the client-side handler element (attribute-registry.js:485).
**Auth / BaaS** βÄî `scrml:auth` stdlib module: magic-link / email-verify / password-reset flows + HS256 JWT + JWKS RS256 verification + `generatePassword`. **¬ß20.5 session-establishment (`session.set`/`.destroy`/`.userId`/`.role`/`.get`/`.isAuth`) βÄî LANDED this window**, the write half of the session model (the compiler owns an HttpOnly session cookie, read back for `@currentUser`). See auth.map.md for the full mechanism (both landings: the base primitive + the pass-2 `__Host-`/`session-secure=`/reserved-key hardening).
**Server/client boundary** βÄî inferred, not annotated: importing a server-tagged stdlib module (or DB/crypto/host access) escalates the importing function to server-only (¬ß12.2); referencing `session` also escalates (¬ß20.5). `E-CG-001` is the fail-closed backstop that blocks any protected DB column from reaching the emitted client bundle (acorn-exact scan, ¬ß14.8.9).
**Colorless async βÄî ¬ß13.1/¬ß13.2, ratified S258, Seam-A LANDED this window (GITI-037/GITI-038, S267/S269/S271).** scrml source has no `async`/`await` keywords; the compiler infers async-ness by tracing calls to Promise-returning host primitives (stdlib, or a returned/aliased local peer) and auto-inserts `await`, coloring the enclosing function `async`. Landed this window: (a) the classifier-unification Phase-1 fix (GITI-037 βÄî a plain function calling `safeCallAsync` no longer silently leaks the Promise; unified onto `computeAsyncFnNames`); (b) the Phase-2 async-collection-combinator TRANSFORM (`some`/`every`/`find`/`filter`/`map`/`forEach`/`reduce`/`flatMap` βÄî a callback reaching an async call now compiles through a `_scrml_*Async` runtime combinator instead of leaking a Promise into a sync iterator method; `.sort` stays fail-closed, no async form); (c) the GITI-038 fix βÄî a factory `return`ing a NAMED function expression whose body is async-colored now TRANSFORMS correctly (`return async function name(){awaitβÄΠ}`, the outer factory stays non-async) instead of silently miscompiling βÄî see schema.map.md's `fnExprNode` entry for the AST-level completeness fix this required across ~10 analysis passes; (d) i87 ¬ß13.2 position-invariant auto-await (#87) βÄî the same auto-await now fires one block deep inside `if`/`for`/`while` bodies, not just at the top of a function. Remaining bucket-(c) fail-closed cases (a raw async call in a param default / `.sort` comparator / a not-fully-transformed arrow/const-bound returned closure) are tracked as `g-async-closure-body-raw-no-lower` (deferred, not shipped this window).
**Writer-ownership Axiom βë† βÄî ¬ß5.5.3/¬ß5.5.4, LANDED this window (#81, S268).** A physical DOM surface (a `class`/`style`/`value` attribute or a form control's `.value` property) has AT MOST ONE wholesale reactive writer. Before this: a `class=(expr)`/`style=(expr)`/`value=(expr)` reactive value attribute dropped OUTSIDE `<each>` (0 diagnostics, silently missing from the emitted HTML). Now: a sole wholesale writer emits correctly (fixes #81); a wholesale writer CONTENDING with another writer on the same surface (e.g. `class=(expr)` alongside `class:name=` token toggles) fires `E-ATTR-WRITER-CONFLICT` naming both sites instead of one silently clobbering the other. Axiom βëΓ (decomposed-surface MERGE, letting two partial writers coexist) is a ruled-but-HELD follow-on, not built.
**Typed API surfaces** βÄî `<api>` (¬ß60, typed EXTERNAL API consumption) vs `<endpoint>` (¬ß61, typed INBOUND endpoint βÄî the serve-side mirror). ¬ß61/¬ß64.9 compose: an `<endpoint>` inside a `kind="tool"` program needs a `serve=` listener or it is unhosted (E-TOOL-ROUTE-NEEDS-SERVE).
**Linear types** βÄî `lin` (¬ß35) + the `~` pipeline-accumulator keyword (¬ß32) for exactly-once-consumed values.
**Input state** βÄî `<keyboard>`/`<mouse>`/`<gamepad>` (¬ß36) are LIVE-READ, not reactive-subscribed; `<#id>.field` inside `${}` renders once at mount (W-INPUT-STATE-MARKUP-NONREACTIVE steers to the `@cell` bridge).
**Build/deploy asset addressing βÄî ¬ß47.9.8.** On the `scrml build` deploy path, per-page client bundles and per-page CSS are content-addressed (FNV-1a hash spliced before the extension); both serve paths send `immutable`/`no-cache`+ETag cache headers accordingly. Not a language-surface feature βÄî a compiler build-output contract. See build.map.md.

## Business Invariants (language axioms, not app rules)
- `null`/`undefined` do not exist in scrml source, in ANY position (¬ß42). Absence is `not`.
- Specificity is deleted under ¬ß65: an unconditional same-property overlap on a provably-shared element is a COMPILE ERROR (E-STYLE-CONFLICT), never a silent cascade pick.
- A protected DB column can never reach the client bundle βÄî fail-closed, acorn-exact (E-CG-001, ¬ß14.8.9).
- Server/client execution boundary is INFERRED from usage (import/API surface, or `session` reference), never author-annotated.
- Match/enum coverage must be exhaustive at compile time (¬ß18) βÄî no runtime default-arm fallthrough.
- `async`/`await` are not scrml keywords (¬ß19.9.8/¬ß13.1) βÄî async is an inferred/desugared codegen concern, not surface syntax; a returned function EXPRESSION carries its own independent async coloring from its enclosing factory (GITI-038, this window).
- Auth tokens (magic-link/verify/reset) are single-use (get-then-delete) and namespace-scoped per purpose βÄî a reset token cannot replay as a magic link.
- A shell SHALL contain at most one `<outlet>` (¬ß20.8) βÄî no nested/multiple outlets in V1.
- A `serve=` headless tool target has NO cookie-session auth surface βÄî fail-closed rejected, not silently unguarded (¬ß64.9).
- An unresolved server-only `scrml:*` re-export's async classification defaults to async (fail-closed), never sync (the STDLIB-EXPORT-SEED backstop).
- A `@`-sigil is required at a CSS value use site to reference a `<theme>` token or a reactive cell (¬ß65.3.2/¬ß25) βÄî a bare identifier is always a literal CSS value, never a token/cell reference.
- Cache immutability for a build artifact is decided by EXACT set membership in the compiler's own content-addressed output, never by a filename-shape guess (¬ß47.9.8).
- Each physical DOM surface has at most ONE wholesale reactive writer (¬ß5.5.3/¬ß5.5.4, Axiom βë†, NEW this window) βÄî a second wholesale writer on the same surface is a compile error, not a silent clobber.
- `session` is reachable ONLY from a server-escalated function body (¬ß20.5, NEW this window) βÄî a client-side/top-level `session` reference is E-SCOPE-012, not an undeclared-identifier fall-through; the client-side equivalent is the `@session` projection.
- `session.set("csrfToken", βÄΠ)` is a reserved-key write and is rejected at compile time when literal (¬ß20.5.1, NEW this window) βÄî the compiler alone mints the CSRF synchronizer token; a dynamic-key write is refused at runtime as a no-op.

## Domain Events (compiler-pipeline analogs)
`RowChange` βÄî synthesized per ¬ß38.13 watched-table row mutation (INSERT/UPDATE/DELETE), dispatched client-side via the `__change` frame to `<onchange>` handlers.
Engine variant transition βÄî an `<engine>` cell's `rule=`-governed state change, optionally observed via `<onTransition>`/`<onTimeout>`/`<onIdle>`.
Soft navigation βÄî a route swap into `<outlet>` (fetch βÜ£ swap βÜ£ hydrate βÜ£ transition), NOT a shell re-boot (¬ß20.8.2); reachable both by explicit `navigate()` calls AND by an ordinary `<a href>` click (link-boost).
Theme-switch reflection βÄî a `<theme for=@cell>` binding re-runs a `_scrml_effect` on every `@cell` change, writing the active variant tag onto `<html data-scrml-theme-<cell>>` (¬ß65.6) βÄî a page-wide CSS-selector flip with zero re-render.
Diagnostic emission βÄî every pipeline stage (BS/TAB/CE/TS/CG, see dependencies.map.md) emits `{code, message, severity, span}` records partitioned into `result.errors`/`result.warnings` (see error.map.md).

## Aggregates (structural elements that own a bounded body)
`<engine>` in compiler/src/ast-builder.js βÄî owns its variant-graph rules + state-child bodies (EngineDeclNode.bodyChildren).
`<channel>` in compiler/src/ast-builder.js βÄî owns its watches= table binding + message/broadcast handlers (ChannelDeclNode).
`<theme>` in compiler/src/theme-body-parser.ts βÄî owns its named-token bindings + `.Variant`/`@media` re-bind blocks (¬ß65.6); its emission is owned by `compiler/src/codegen/emit-theme-reset.ts`.
`<schema>` βÄî owns its table/column DDL surface (¬ß39), consumed by protect-analyzer.ts for the ¬ß14.8.9 protect-floor.
`<outlet>` βÄî owns the swappable route-content region inside a `<program>` shell (¬ß20.8.1); NOT a dedicated AST node (see schema.map.md).
A returned function-expression closure (`return function name(){βÄΠ}`, GITI-038, NEW this window) βÄî owns its own body's scope/type/async analysis independent of its enclosing factory (`ReturnStmtNode.fnExprNode`, see schema.map.md).
