## Summary

`scrml dev <directory>` recursively gathers **every** `.scrml` under the input
dir with no `node_modules` skip and no symlink guard. In any real adopter
project where `node_modules/<pkg>` is a `bun link` symlink to a source repo, the
scanner walks the entire linked tree and tries to compile thousands of stray
sources — the dev server never finishes the initial compile and never listens.

## How scrml-site hit it (the real adopter path)

scrml-site consumes scrmlTS as a **linked dependency** (`bun link scrmlts`), per
the S154 ratification (dependency, not vendored copy). That makes
`scrml-site/node_modules/scrmlts` a symlink → the whole sibling `scrmlTS` repo.
Running `scrml dev .` (repo root) then gathers scrmlTS's `stdlib/`, `samples/`,
`examples/`, etc. Observed: the process stays alive compiling
`node_modules/scrmlts/samples/compilation-tests/*`, `.../stdlib/auth/jwt.scrml`,
… and never reaches `[dev] Serving`.

## Root cause (exact)

`compiler/src/api.js` → `scanDirectory(dirPath)` (≈line 86):

```js
function walk(dir) {
  const entries = readdirSync(dir);
  for (const entry of entries) {
    const fullPath = join(dir, entry);
    const stat = statSync(fullPath);      // follows symlinks
    if (stat.isDirectory()) walk(fullPath);          // ← descends into node_modules + symlinked repos
    else if (entry.endsWith(".scrml")) results.push(fullPath);
  }
}
```

Two issues: (1) no skip for `node_modules` / `.git` / `dist`; (2) `statSync`
follows the symlink so `node_modules/<linked-pkg>` is walked as a real dir.

Note the **watcher** was already hardened against exactly this tree in the
BUG-1 fix (`commands/dev.js` `deriveWatchFiles`, "never touches node_modules or
sibling directories") — but the **gather** (`scanDirectory`) was not, so the
hazard moved upstream from "watch storm" to "compile storm / no-listen".

## Suggested fix (your call on shape)

In `scanDirectory.walk`, skip dot-dirs + `node_modules` + the output dir, and
use `lstatSync` (or `Dirent.isSymbolicLink()` via `readdirSync(dir,
{withFileTypes:true})`) to not follow symlinked directories. e.g.:

```js
if (entry === "node_modules" || entry === ".git" || entry === "dist" || entry.startsWith(".")) continue;
const st = lstatSync(fullPath);
if (st.isDirectory()) walk(fullPath);
```

This is a pure-additive scoping change; it can't broaden what a normal project
compiles (those dirs never held routable `.scrml`).

## What scrml-site did meanwhile (no compiler change needed on our side)

`scripts/serve.sh` now passes the app's own `.scrml` files **explicitly**
(`app.scrml pages/*.scrml components/*.scrml lib/*.scrml`) instead of `scrml dev
.`. That yields the same `inputFiles` set + common-prefix (repo root) as
directory mode would, minus the node_modules pollution — and serves cleanly
(verified end-to-end: `/`, `/dashboard`, `/data/mario/*` static, real `.js.map`
with `x_scrml_kinds`, client provenance bundle). So we're unblocked; this report
is for the upstream fix that makes `scrml dev .` work for adopters generally.
