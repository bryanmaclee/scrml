## The skeleton (RATIFIED)

### 1. Pipeline shape — RE-CUT (forced, not chosen)
The TS 17-stage pipeline → **4 macro phases**: **lex** (single fold) · **parse** (recursive descent — honestly NOT a fold; produces the untyped surface AST) · **analyze** (nanopass micropass sequence) · **lower+emit** (fold; JS target deletes the instruction-selection/regalloc back-half). The re-cut is FORCED: the TS in-place B1–B22 AST decoration (`_record`/`_scope`/`_nodeCounter`) is UNREPRESENTABLE in idiomatic scrml (§45.6 no value identity). The conformance suite (D3) makes the re-cut safe (pins (a)-codes + (b)-runtime; AST/JS-shape/message = impl freedom).

### 2. IR model — small fixed Elm-shaped catalog
`string → [Token] → FileAst (untyped surface) → TypedProgram → CoreProgram → FileOutput`. ~5 IRs (NOT nanopass's ~20 — no boilerplate-autogen in scrml). **REUSE the existing canonical `ast-expr.scrml`/`ast-stmt.scrml` `Expr`/`Stmt` enums.** **TypedProgram = stable AST + immutable id-indexed SIDE-TABLES** (scopes/types/depGraph/…), NOT in-place decoration (forced by §45.6/§59.5). The parser mints an explicit **`NodeId`** (purely-threaded counter replacing the TS global `_nodeCounter`).

### 3. The analyze phase / resolver — **HYBRID** (the debate ruling)
- **Local resolution + typing + cycle-detection = Pole B** — an env-threading catamorphism (Algorithm-W: thread `(scope-env, type-env)`, names resolve at point-of-use, dependency-order IS the fold order) + **local binding-group SCC** for cycles (Mark-Jones/GHC `Note [Dependency analysis]`). Cycles are detected as **Tarjan back-edges (a control-flow event), NEVER stored as a value** → the §59.5 "cyclic-value" keystone EVAPORATES. Collapses B7+B10+B16+§48.6.4 (mutual-recursion + the 3 forbidden-cycle classes) into one per-group fold.
- **The whole-program analyses = Pole A's form** — §31.5 transitive-reads · §40.9 reachability · §47 per-route splitting are genuinely GLOBAL (both poles conceded) and consume an **explicit id-indexed VALUE-graph** (a `[NodeId: DGNode]` + `[DGEdge]` side-table — finite/acyclic, §59.5-respecting; the GHC `stronglyConnComp`-over-`(node,key,[key])` form). FBIP's alias fold is a **4th consumer** (Fold A projects the §40.9 def-use edges) → the graph is a genuinely SHARED artifact, not Pole-A scaffolding.
- **Net:** thread-fold the local, project ONE id-indexed graph side-table for the global. = exactly the pipeline dive's "analyze micropasses → id-indexed depGraph side-table," reached independently. Pole A's "graph-as-spine" REJECTED; Pole B's "never materialize" REJECTED.

### 4. FBIP rung-3 — inferred reuse + differential harness
Three folds over the typed IR: **Fold A** fwd alias+escape (a projection of the §40.9 DG def-use edges) · **Fold B** bwd liveness (generalizes `lin`'s backward walk to all bindings) · **Fold C** per-seam `InPlace|Clone` (InPlace iff dead ∧ unique ∧ non-escaping). Uniqueness = LIVENESS (no value identity §45.6 + COW ⇒ alias set is read off the def-use bind graph, not a points-to lattice). `lin` is necessary-but-INSUFFICIENT (gives Fold B, not Fold A's alias half) — operationalizes the S225 middle-rung retirement. `reuse` is a threaded IR field (codegen re-parses → must survive). Conservatism is the floor (any uncertainty → Clone; clone-always = correctness). Reactive `@` cells ALWAYS clone (§6.5.3 carve-out — coincides exactly with the non-reactive FBIP target). **Soundness gate = a differential harness** (compile twice `--reuse=off/on` → the W3 conformance adapter → assert byte-identical {dom,state,codes}); STRONGER than CSmith/EMI (in-place purity ⇒ GUARANTEED equivalence ⇒ zero false-positives). Coverage 70-90% is a code-shape ESTIMATE (workload-dependent; report per-corpus); coverage is a speed dial, never a correctness dial.

### 5. Conformance-driven build — near-autonomous
Loop-until-green (SWE-bench shape: red case = `FAIL_TO_PASS` work-item · green-set = `PASS_TO_PASS` regression guard). Oracle = the already-built `conformance/run.ts` adapter (boolean). **impl#2 = ONE net-new adapter file** plugged into the same runner+corpus (test262 "one suite, every engine self-declares"). The suite **bisects at the typer/codegen seam**: lexer→token-diff · parser→the already-built within-node parity classifier · typer→the (a)-codes half · codegen→the (b)-runtime half = full conformance. Termination = case→wave→build (whole 1.0-labeled suite green = impl#2 conformant = Road-B/W5 cleared). **"Near-autonomous build" is LITERALLY TRUE** — the only non-automatable points are the two human bookends: decide-architecture (this RULING) + author-the-contract (V1). Maps onto mitosis: language-PA owns the contract + gap-rulings; the compiler-PA daughter runs the implement-to-green fleet. Autonomy ∝ suite-completeness → building V1 grows the contract → every future compiler build is more autonomous (the work compounds, made mechanical).
